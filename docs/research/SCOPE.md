# Research Scope

## Topic: AI-Augmented Development — Techniques & Tools for Maximum Developer Productivity and Application Quality
## Purpose: Build a living knowledge base that helps a solo/small-team developer ship top-of-the-line applications faster by leveraging AI agents, modern tooling, and emerging dev patterns. Focus on what's practical and proven, plus what's new and worth adopting.

## Key Questions:
1. **Agent-assisted dev workflows**: What are the best current patterns for using AI agents (Claude, Cursor, Copilot, Aider, etc.) to accelerate real development? What workflows actually save time vs. create busywork?
2. **Multi-agent / swarm architectures**: How are developers using multi-agent systems to handle complex tasks (code gen, testing, review, deployment)? What frameworks exist and which are production-ready?
3. **MCP, skills, plugins**: What are the most effective ways to extend AI dev tools with custom context, tools, and integrations? What MCP servers, plugins, and skill patterns are developers actually using?
4. **AI-native application patterns**: What architectural patterns are emerging for building apps that are designed around AI from the ground up (AI-first UX, agent backends, RAG, etc.)?
5. **Cutting-edge techniques**: What's new in 2025-2026 that changes how a developer should work? (new frameworks, paradigm shifts, tools gaining traction)
6. **Quality at speed**: How are the best teams shipping high-quality code fast with AI — testing strategies, review patterns, deployment automation?

## Out of Scope:
- Model training / fine-tuning
- AI safety/alignment theory
- Enterprise procurement / pricing comparisons
- Academic ML research unrelated to building software

## Prior Knowledge:
- User is an active developer already using Claude (Cowork, skills, plugins, MCPs)
- Familiar with agent concepts — wants to go deeper on what works in practice
- Goal: maximize personal productivity and output quality, not just learn theory
