**1. Superpowers — Agentic Skills Framework for Coding Agents**

An open-source framework by Jesse Vincent that enforces structured workflows (design → planning → TDD → debugging) on AI coding agents, turning them from reactive assistants into autonomous developers that can work for hours unsupervised. Gained 1,867 GitHub stars in a single day (March 16) and 27K total in three months, with support for Claude Code, Cursor, and Codex.

- **Impact:** Transformative — this is the missing methodology layer that makes coding agents reliable enough for real production work.

**3. Simon Willison's Agentic Engineering Patterns Guide — Practical Patterns for Working with Coding Agents**

Published March 4 and actively updated (12 chapters as of this week), this guide codifies practical patterns for effective agent-assisted development: red/green TDD adapted for agents, linear walkthroughs for code comprehension, and the principle of "hoarding things you know how to do" as reusable agent context. Based on extensive real-world usage across Claude Code and other tools.

- **Why it matters:** The first serious attempt to document *how* experienced developers actually get good results from coding agents — moves beyond "tips and tricks" into repeatable engineering methodology.
- **Impact:** High — immediately actionable patterns that improve agent output quality today.
- **Source:** [simonwillison.net/guides/agentic-engineering-patterns](https://simonwillison.net/guides/agentic-engineering-patterns/) | [Substack announcement](https://simonw.substack.com/p/agentic-engineering-patterns)

## Evening Update — Additional Findings

### 4. Claude Code v2.1.77–2.1.79 Ships This Week: `/remote-control`, Plugin State, Memory Gains
Three Claude Code versions dropped March 17–18:
- **v2.1.79** (today): `/remote-control` bridges VSCode sessions to claude.ai/code for browser/phone continuation; AI-generated session tab titles; `--console` flag for API billing auth; **18MB startup memory reduction**
- **v2.1.78**: `StopFailure` hook event for API errors; `${CLAUDE_PLUGIN_DATA}` variable for **plugin persistent state**; new frontmatter fields (`effort`, `maxTurns`, `disallowedTools`) for plugin-shipped agents; security fix for silent sandbox disable
- **v2.1.77**: Default max output tokens for Opus 4.6 raised to **64k**; `allowRead` sandbox filesystem setting; `/copy N` index support

**Impact**: Plugin persistent state and agent frontmatter controls significantly improve plugin/skill authoring. `/remote-control` enables cross-device development continuity.

Source: [Claude Code Changelog](https://code.claude.com/docs/en/changelog)

### 5. AI-Generated Code Introduces Vulnerabilities in 87% of Pull Requests
DryRun Security's March 11 report tested Claude, Codex, and Gemini across 38 scans of full application generation. Results: 143 vulnerabilities surfaced, with **87% of PRs introducing at least one flaw**. CrowdStrike separately identified hidden vulnerability patterns unique to AI-generated code. OpenAI launched Codex Security — an AI security agent that scanned 1.2M commits and found 10,561 high-severity issues.

**Impact**: Security review for AI-generated code is no longer optional. Teams need automated security scanning in CI/CD pipelines specifically calibrated for AI code patterns.

Sources: [DryRun Security Report](https://digiconasia.net/news/ai-coding-tools-introduce-security-flaws-in-87-of-pull-requests-report) · [CrowdStrike Research](https://www.crowdstrike.com/en-us/blog/crowdstrike-researchers-identify-hidden-vulnerabilities-ai-coded-software/) · [OpenAI Codex Security](https://thehackernews.com/2026/03/openai-codex-security-scanned-12.html)
