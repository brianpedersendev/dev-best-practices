# AI-Augmented Development Research

## Purpose
A living knowledge base to help Brian ship top-of-the-line applications faster using AI tools, agents, and modern dev patterns. Everything here should be **practical and actionable** — no filler, no theory-for-theory's-sake.

## Folder Structure
```
AIResearch/
├── CLAUDE.md              # This file — project instructions
├── GETTING-STARTED.md     # Entry point — top takeaways, guide links, quick start checklist
├── INDEX.md               # Master directory of all research (always keep updated)
├── project-research.skill # Installable skill — idea vetting → research → plan pipeline
├── project-scaffold.skill # Installable skill — generates AI tooling scaffold for new projects
├── DailyBriefing/         # Auto-generated daily briefings on AI dev trends
├── docs/
│   ├── research/          # Raw research outputs (SCOPE.md, SYNTHESIS.md, etc.)
│   └── topics/            # Individual topic deep-dives as standalone .md files
└── references/            # Links, bookmarks, and source lists
```

## Rules for Adding Content
1. **Every new finding goes in INDEX.md** with a one-line description and link to the full file
2. **Be specific** — "Use X tool with Y pattern to achieve Z" beats "X is useful"
3. **Include sources** — URLs, docs, repos for every claim
4. **Date everything** — prefix findings with discovery date so we know what's fresh
5. **Flag staleness** — if something is older than 3 months, mark it for re-verification
6. **No duplicates** — check INDEX.md before adding; update existing entries instead

## Quality Bar
- Would this change how I build something? If no, cut it.
- Is this actionable today or within ~3 months? If no, move to a "watching" section.
- Does it have a source? If no, flag as unverified.

## Research Areas (priority order)
1. **Dev workflows** — AI-assisted coding patterns that actually save time
2. **Agent architectures** — Multi-agent/swarm systems for real dev tasks
3. **Tool ecosystems** — MCPs, plugins, skills, and how to extend AI tooling
4. **AI-native app patterns** — Architecture for apps built around AI
5. **Emerging techniques** — New tools and paradigms worth watching (2025-2026)

## When Updating This Folder
- Always read this file first
- Check INDEX.md for what already exists before researching
- After adding anything, update INDEX.md
- Keep entries concise — link to detail files for depth
- **Update GETTING-STARTED.md** whenever adding a new topic guide that would help someone getting started — add a link with a clear one-line description under the appropriate section
